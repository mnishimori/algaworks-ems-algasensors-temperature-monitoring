# Documento de Requisitos de Produto (PRD)

## Título
Endpoints de Monitoramento de Temperatura (Básico)

## Visão Geral
Este PRD define os requisitos básicos de API para o serviço de Monitoramento de Temperatura, suportando listagem de histórico de temperaturas, configuração de alertas e ativação/desativação do monitoramento. O escopo é intencionalmente mínimo (YAGNI) e baseado no documento de design fornecido.

## Declaração do Problema
Empresas de monitoramento precisam de uma forma confiável para listar registros de temperatura, gerenciar limites de alerta e habilitar ou desabilitar o monitoramento de sensores instalados em ambientes como galpões, câmaras frias e locais abertos. Atualmente, não há um contrato de API definido para essas operações básicas.

## Objetivos
- Fornecer endpoints básicos de API para histórico de temperaturas, configuração de alertas e ativação/desativação do monitoramento.
- Garantir que a retenção de dados de temperatura seja permanente.

## Fora do Escopo
- Análises avançadas, previsões ou detecção de anomalias.
- Fluxos de trabalho complexos de alertas (escalas, políticas com múltiplos destinatários, agendamento).
- Políticas de exclusão ou expiração de dados.

## Usuários e Stakeholders
- Usuários: Funcionários de uma empresa de monitoramento de temperatura que opera sensores em galpões, armazenamento frio e ambientes externos.
- Stakeholders: Product owners e equipes de backend do microsserviço temperature-monitoring.

## Premissas
- Identificadores de sensores já são provisionados por outros serviços.
- A ingestão de dados de temperatura é tratada em outro local e está disponível para listagem.

## Requisitos Funcionais
RF-01. O sistema deve fornecer um endpoint para listar registros de temperatura de um sensor.
RF-02. O sistema deve fornecer um endpoint para recuperar a configuração atual de alerta de um sensor.
RF-03. O sistema deve fornecer um endpoint para atualizar a configuração de alerta de um sensor.
RF-04. O sistema deve fornecer um endpoint para excluir a configuração de alerta de um sensor.
RF-05. O sistema deve fornecer um endpoint para recuperar detalhes de monitoramento de um sensor.
RF-06. O sistema deve fornecer um endpoint para habilitar o monitoramento de um sensor.
RF-07. O sistema deve fornecer um endpoint para desabilitar o monitoramento de um sensor.
RF-08. O sistema deve reter registros de temperatura indefinidamente.

## Escopo da API (do contexto de design)
- GET /api/sensors/{sensorId}/temperatures
- GET /api/sensors/{sensorId}/alert
- PUT /api/sensors/{sensorId}/alert
- DELETE /api/sensors/{sensorId}/alert
- GET /api/sensors/{sensorId}/monitoring
- PUT /api/sensors/{sensorId}/monitoring/enable
- DELETE /api/sensors/{sensorId}/monitoring/enable

## Métricas de Sucesso
- Todos os endpoints retornam respostas esperadas para IDs de sensor válidos.
- Usuários podem recuperar histórico de temperatura e gerenciar configuração de alerta sem intervenção manual.

## Questões em Aberto
- Nenhuma no momento.
